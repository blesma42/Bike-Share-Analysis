Introduction

About the Company

In 2016, Cyclistic launched a successful bike-share offering. Since then, the program has grown to a fleet of 5,824 bicycles that are geotracked and locked into a network of 692 stations across Chicago. The bikes can be unlocked from one station and returned to any other station in the system anytime.

About the Project

Cyclists finance analysts have concluded that annual members are much more profitable than casual riders. Although the pricing flexibility helps Cyclistic attract more customers, Management believes that maximizing the number of annual members will be key to future growth. Rather than creating a marketing campaign that targets all-new customers, Management believes there is a very good chance to convert casual riders into members. Management notes that casual riders are already aware of the Cyclistic program and have chosen Cyclistic for their mobility needs.

Leading Questions
  • How do regular users (members) and casual users differ in their use of bikes?
    • Do members drive during different days or times?
    • Do members dirve longer?
    • Do members drive with different like types?
  • How could causal users motivated to become members?

Cleaning and Transformation

Because cleaning the data could remove large amounts of it, it might be interesting, how many entries are removed by each step. For this reason a function is generated, which will calculate the number of rows removed from the data frame since the function was last called. number_of_rows was initiated during the reading of he data frames and is currently 5,595,063.

removed_rows <- function(){
  new_number_of_rows <- nrow(df)
  removed_rows <- number_of_rows-new_number_of_rows
  number_of_rows <<- new_number_of_rows
  print(paste('Removed rows:', format(removed_rows, big.mark=',')))
}

Here are several issues noticed, like a lot of empty values, or that start and end date are not in a date time format. To deal with this, empty values are replaced by NA before filtering the data. For the filters its assumed that either a stations name or id is enough to identify the station, therefore at least on of these parameters must be given for both start and end station. Additionally only rides are kept, where both start time and end time are given. Finally the number of removed rows is checked by calling the previously written function removed_rows().

df[df==''] <- NA

#removes rows where no start station or end station is given or the start or end date is missing
df <- df %>%

  filter_at(vars(end_station_name, end_station_id), any_vars(!is.na(.))) %>%
  filter_at(vars(started_at, ended_at), all_vars(!is.na(.)))

Conclusion

The analysis provided some insights into the differences between the user types. However, there are still a lot of unknown elements. for example, its unknown, how much distance the trips covered. Even though the data provided the start and end coordinates of a trip, thats not sufficient to calculate the distance driven. In the worst case, a person makes a long bike trip but returns the bike in the end to the start station, and therefore wouldnt cover any ground. In addition, the coordinates only allow calculation of the distance as the crow flies, but not as a cyclist would actually drive. Also, because of the lack of knowledge about the city structure, geographic patters cant be analysed. E.g. if the morning trips start in residential districts and end in industrial areas, or if people from certain districts use the bike more often than others. Another uncertainty are the individual users. Because the data didnt contain a user id or something similar, its not possible to tell, how many users are there, or if they have some repeating pattern, e.g.every day at the same time the same trip. Such patterns would indicate a trip to work or something similar. This could be interesting for targeting specific groups, e.g.casual users which would benefit from a membership.

Based on this analysis I conclude, that there is not enough information about the users to launch a targeted advertisement campaign Instead, I recommend another analysis with the mentioned elements above, or a survey, were customers are asked, why they choose a member ship or why not, if they are aware of the benefits, and other similar questions.

This amount could indicate an issue with the system, which collects the data and writes it into the data base. Also, the unequal number of station names and ids should not have happened, and could easily be solved with a relational database.
